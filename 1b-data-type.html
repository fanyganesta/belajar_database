C. DECIMAL ‚Äî Angka Presisi (WAJIB untuk Uang)

Apa itu DECIMAL?

DECIMAL menyimpan angka dengan presisi tetap.

Format:

    DECIMAL(total_digit, decimal_digit)

Contoh:

    DECIMAL(15,2)

Artinya:

  • Total 15 digit
  • 2 digit di belakang koma



Kapan Wajib Pakai DECIMAL?

  • Harga
  • Uang
  • Pajak
  • Diskon
  • Financial calculation

üìå Tidak ada kompromi untuk ini.



Kenapa BUKAN FLOAT / DOUBLE?

Karena:

  • Floating point error
  • Pembulatan tidak konsisten
  • Fatal untuk finance




Contoh error klasik:

Contoh Penggunaan yang Benar

Kesalahan Fatal di Dunia Nyata

‚ùå Pakai INT untuk rupiah
‚ùå Pakai FLOAT untuk pajak

Kenapa Date & Time Itu Rawan?

A. DATE ‚Äî Tanggal Tanpa Waktu

üìå Tidak ada jam
üìå Tidak ada menit
üìå Tidak ada zona waktu



Kapan Pakai DATE?

Gunakan DATE untuk:

  • Tanggal lahir
  • Tanggal invoice
  • Tanggal jatuh tempo
  • Periode bisnis
Contoh:

    birth_date DATE 

    invoice_date DATE 

    due_date DATE


DATE BUKAN untuk:

‚ùå Log aktivitas
‚ùå Timestamp transaksi

B. DATETIME ‚Äî Tanggal + Jam (Netral Zona)

Apa itu DATETIME?

DATETIME menyimpan tanggal dan waktu, tanpa peduli zona waktu.

Format:

    YYYY-MM-DD HH:MM:SS

üìå Tidak otomatis konversi zona
üìå Disimpan apa adanya



Kapan Pakai DATETIME?

Gunakan DATETIME untuk:

  • Waktu transaksi
  • Jadwal
  • Event bisnis
  • Data historis


Contoh:

    created_at DATETIME 

    paid_at DATETIME 

    scheduled_at DATETIME


Kelebihan DATETIME

  • Stabil
  • Tidak tergantung timezone server
  • Aman untuk laporan historis


C. TIMESTAMP ‚Äî Waktu Absolut (Zona Waktu Aktif)

Apa itu TIMESTAMP?

TIMESTAMP menyimpan waktu absolut (UTC) dan otomatis dikonversi ke timezone server.

üìå Disimpan sebagai UTC
üìå Ditampilkan sesuai timezone



Kapan Pakai TIMESTAMP?

Gunakan TIMESTAMP untuk:

  • Log sistem
  • Audit trail
  • Event real-time
  • Tracking aktivitas user
