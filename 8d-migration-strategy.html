D. Migration Strategy (Versi Pemula, Tapi Profesional)



A. Apa itu Migration? (Definisi Paling Sederhana)

Migration = catatan perubahan database dari waktu ke waktu. Kalau diibaratkan: Database = bangunan Migration = riwayat renovasi bangunan Contoh: hari ini nambah kamar besok ganti pintu lusa nambah lantai Semua perubahan itu dicatat, bukan diingat di kepala.

B. Kenapa Migration Itu Penting?

Tanpa migration: lupa sudah ngubah apa laptop A beda dengan server B error muncul tapi nggak tahu sebabnya Dengan migration: semua perubahan terurut bisa diulang bisa dijelaskan ke orang lain
